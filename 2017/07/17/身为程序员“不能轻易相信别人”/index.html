身为程序员“不能轻易相信别人”

发表于 2017-07-17




经过周五周六晚上的熬夜，终于把这个“github+hexo”的博客搭建好了，我很是激动啊。但是我忽然想到了一个问题，那么我要用它来干些什么呢？写文章？写日记？还是写一些惯有技术方面的事情？

自己技术那么渣怎么可能写出什么大道理的技术性文章，那么自己就记一些总结好了。把自己犯的错误记下来，好好总结，反思一下，顺便当一个日记平台好了。

首先我想说的是上周五发生的事情，我真的是很无语，既生气又生不起来气，憋在心里很难受。我遇到了一个很简单的技术问题，就是我打了断点，但是它没debug，反而弹出了图中的错误，这种错误解决方法很简单啊：

preferences-java-complier下有一个Add line number attributes to generated class files，勾上它。

我各种百度，找各种解决问题，我当时都急了，怎么就是不好使呢？是哪里出问题了啊？我就想，快下班了，又要加班啊、svn上导的项目又出问题啦？天呐。。。我还想他是不是又不编译了，因为我之前在这个方法里的输出又加了一个输出语句，但是我添加的没有输出出来。我就各种脑洞大开，最后求救了朋友，还是人家细心啊。。我真尴尬，之所以不进debug，是因为项目根本就没有走这个方法。没用到。我之前还问写这个方法的人，是不是用的这个方法，他跟我说是！我就这么相信了别人。再加上自己着急慌乱，还有些渣，那更解决不了啊。。。

所以希望自己遇事不要慌，能沉着冷静。改掉毛毛躁躁的个性。不求自己有多好，但求自己无愧。
